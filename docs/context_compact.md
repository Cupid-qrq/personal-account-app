# Context Compact - 账本管理系统

更新时间：2026-04 v0.4修复版（DuplicateElementId bug 解决）

## 核心使命

实现月度账本导入→清洗→分析→可视化，支持家庭多用户、移动端优化、云端部署。

## 上一版的问题（v0.4 初版）

1. **DuplicateElementId 错误**: HTML div 包裹 + 循环按钮产生重复 key
2. **登录界面框显示不全**: 输入框被 HTML 架构破坏
3. **登陆后全是红色报错**: Streamlit 组件冲突导致

## v0.4修复版的解决方案

### 😀 登录页面重与
- 移除 HTML `<div>` 包裹，使用纯 Streamlit 组件
- 简化输入流程：用户名 → 密码 → 登录
- 添加演示账号提示卡
- 移除复杂布局问题

### 🔌 按钮 key 冲突修复
- 旧问题: 循环中的按钮产生重复 key（`key=f"cat_{category}"`）
- 新解决: 用 `st.selectbox("分类", options=[...])` 替代循环按钮
- 效果: 完全消除 DuplicateElementId 错误

### 🎨 代码精简
- 删除所有重复的 CSS 定义（旧文件有 3 份样式块）
- 保留关键美观样式（毛玻璃卡片、渐变、响应式）
- 减少文件容量，提升加载速度

- 代码语法检查: ✅ 通过

## 核心目标

- 对月度账单 CSV 进行导入、清洗、归档与高级可视化分析。
- 支持多用户认证：管理员上传数据，家人访客仅查看。
- 确保移动端体验流畅，自适应各屏幕尺寸。
- 部署到 Streamlit Cloud，全球用户无障碍访问。

## 已实现能力（v0.4）

### 🔐 身份认证模块（NEW）
- 登录页面：用户名 + 密码 + 演示账号提示
- 三账户体系：cupid(admin) + dad(viewer) + mom(viewer)  
- 权限粒度：admin=可上传 + 查看，viewer=仅查看
- 会话管理：`st.session_state` 存储用户身份，页面刷新保留状态

- 字体自适应：指标值/标签在移动端自动缩放、保持可读性
- 卡片自适应：Padding/高度/边距在小屏自动调整
- Plotly 图表：高度动态设定，防止超宽
- 触摸优化：按钮尺寸增大（>44px），点击区域友好

## 关键设计决策

- **深色主题**：减少眼疲劳，更科技感
- **毛玻璃卡片**：现代 UI 审美
- **表单化标签页**：清晰信息分层
- **数据驱动评分**：量化消费健康度
- **预警阈值**：标准差法则实现智能异常检测
- **Session State**：记录用户交互状态
- **多部署方案**：满足不同场景需求
