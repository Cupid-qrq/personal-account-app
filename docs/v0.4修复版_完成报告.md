## 🐛 修复详情

### 问题 1: StreamlitDuplicateElementId 崩溃
**症状**: 登陆后全是红色错误，应用无法使用  
**根因**: 循环生成的按钮产生重复 key
```python
# ❌ 旧代码（有bug）
for idx, (_, row) in enumerate(cat_df.iterrows()):
    category = row["分类"]
    with cols[idx % len(cols)]:
        if st.button(f"{category}...", key=f"cat_{category}"):  # 同名分类重复!
```

**修复方案**: 用 selectbox 替代循环按钮
```python
# ✅ 新代码（无bug）
cat_list = [""] + cat_df["分类"].tolist()
selected_cat = st.selectbox("分类", cat_list, key="cat_select")
```

---

### 问题 2: 登陆框显示不出来
**症状**: 输入框、按钮无法正常显示  
**根因**: HTML `<div>` 包裹破坏了 Streamlit 组件结构
```python
# ❌ 旧代码（有bug）
st.markdown('<div class="login-card">', unsafe_allow_html=True)
st.markdown("# 💳 账本系统")
username = st.text_input(...)  # 组件在 HTML 外
st.markdown('</div>', unsafe_allow_html=True)
```

**修复方案**: 移除 HTML，使用纯 Streamlit 组件
```python
# ✅ 新代码（无bug）
col1, col2, col3 = st.columns([1, 1.2, 1])
with col2:
    st.markdown("## 💳 账本系统")
    username = st.text_input("用户名", key="login_user")
    password = st.text_input("密码", type="password", key="login_pass")
    if st.button("🔓 登录", use_container_width=True, key="do_login"):
```

---

### 问题 3: CSS 样式重复加载混乱
**症状**: 页面加载缓慢，样式冲突  
**根因**: CSS 样式块重复定义 3 次
```python
# ❌ 旧代码
st.markdown("""<style>...</style>""")  # 第 1 次
# ... 中间 1000 行代码 ...
st.markdown("""<style>...</style>""")  # 第 2 次
# ... 再来 200 行 ...
st.markdown("""<style>...</style>""")  # 第 3 次！
```

**修复方案**: 合并为单一样式块
```python
# ✅ 新代码
st.markdown("""<style>
    /* 所有样式合并在这里 */
</style>""", unsafe_allow_html=True)
```

## 🎓 技术总结

### 问题分类
- **架构问题**: HTML 与 Streamlit 组件混合
- **命名问题**: 按钮 key 重复导致冲突
- **代码维护**: 样式重复定义 3 次
- **最佳实践**: 忽视了 Streamlit 组件隔离原则

### 解决方案模式
1. **纯粹性**: 避免混合 HTML 和 Streamlit
2. **唯一性**: 确保所有 key 唯一
3. **单一性**: CSS 样式集中管理
4. **模块化**: 函数拆分，逻辑清晰

### 收获
- Streamlit 组件必须在 context 管理器内
- key 冲突是 DuplicateElementId 的常见原因
- 循环生成组件时需要特别注意命名
- HTML 美化应该用 CSS class 而非嵌套
